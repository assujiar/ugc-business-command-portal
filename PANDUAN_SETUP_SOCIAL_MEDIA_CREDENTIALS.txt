================================================================================
  1. APA SAJA YANG PERLU DISIAPKAN?
================================================================================

Untuk setiap platform, kalian perlu menyiapkan beberapa data berikut:

  Data yang Dibutuhkan        | Penjelasan Singkat
  ----------------------------|----------------------------------------------------
  Access Token                | "Kunci akses" agar sistem bisa membaca data
                              | analytics akun social media kalian.
  Refresh Token               | "Kunci cadangan" agar sistem bisa memperbarui
                              | Access Token secara otomatis tanpa login ulang.
  Account ID                  | ID unik akun/halaman/channel kalian di platform.
  Client ID / Client Key      | Identitas aplikasi yang dibuat di developer portal.
  Client Secret / App Secret  | Password rahasia aplikasi (JANGAN SHARE sembarangan).

Tidak semua platform butuh semua data di atas. Lihat panduan per platform.


================================================================================
  BAGIAN A: TikTok
================================================================================

SYARAT:
  - Punya akun TikTok Business

APA YANG HARUS DILAKUKAN:

  1. Daftar di TikTok for Developers
     - Buka: https://developers.tiktok.com/
     - Klik Sign Up, gunakan akun TikTok Business
     - Verifikasi akun

  2. Buat Aplikasi Baru
     - Masuk ke: https://developers.tiktok.com/apps/
     - Klik "Create App"
     - Isi nama aplikasi, contoh: "UGC Portal Analytics"
     - Isi deskripsi singkat (untuk apa aplikasi ini)
     - Upload icon/logo
     - Pilih fitur: Login Kit, User Info (Basic & Stats), Video List
     - Submit dan tunggu di-approve oleh TikTok (biasanya 1-3 hari kerja)

  3. Setelah Disetujui, Catat Data Berikut
     - Buka halaman app > bagian "Configuration"
     - Catat:
         Client Key    = ........................ (catat di Tabel Ringkasan)
         Client Secret = ........................ (catat di Tabel Ringkasan)

  4. Hubungkan Akun TikTok
     - Minta bantuan Developer untuk menjalankan proses otorisasi
     - Kalian akan diminta login TikTok dan klik "Izinkan"
     - Setelah berhasil, Developer akan mendapat:
         Access Token   = (otomatis didapat)
         Refresh Token  = (otomatis didapat)
         Account ID     = (otomatis didapat, berupa Open ID)

DATA YANG HARUS DISERAHKAN KE DEVELOPER:
  [x] Client Key
  [x] Client Secret
  [x] Akses login ke akun TikTok Business (untuk proses otorisasi)

CATATAN:
  - Token TikTok berlaku 24 jam, tapi sistem akan refresh otomatis
  - Refresh Token berlaku 1 tahun. Setelah 1 tahun perlu otorisasi ulang


================================================================================
  BAGIAN B: Instagram & Facebook
================================================================================

Instagram dan Facebook dikelola dalam satu tempat (Meta).
Prosesnya saling terkait, jadi dikerjakan bersamaan.

SYARAT:
  - Punya Facebook Page (halaman bisnis, BUKAN profil pribadi)
  - Punya akun Instagram Business atau Creator
  - Akun Instagram SUDAH terhubung ke Facebook Page
    (Cek di: Instagram > Settings > Account > Linked Accounts > Facebook)

APA YANG HARUS DILAKUKAN:

  1. Daftar di Meta for Developers
     - Buka: https://developers.facebook.com/
     - Login menggunakan akun Facebook yang punya akses Admin ke Page
     - Klik "My Apps" > "Create App"
     - Pilih tipe: "Business"
     - Isi nama aplikasi, contoh: "UGC Portal Social Media"
     - Pilih Business Portfolio yang sesuai
     - Klik "Create App"

  2. Tambahkan Fitur
     - Di halaman dashboard aplikasi, klik "Add Product"
     - Tambahkan: "Facebook Login for Business"
     - Tambahkan: "Instagram Graph API" (kalau ada di menu)

  3. Catat Data Aplikasi
     - Klik Settings > Basic di sidebar kiri
     - Catat:
         App ID     = ........................ (catat di Tabel Ringkasan)
         App Secret = klik "Show" lalu catat (catat di Tabel Ringkasan)

  4. Buat Access Token via Graph API Explorer
     - Buka: https://developers.facebook.com/tools/explorer/
     - Di dropdown "Meta App", pilih aplikasi yang baru dibuat
     - Klik "Generate Access Token"
     - Akan muncul popup login, berikan izin berikut:
         Untuk Facebook:
           pages_show_list
           pages_read_engagement
           pages_read_user_content
           read_insights
         Untuk Instagram:
           instagram_basic
           instagram_manage_insights
           pages_show_list
           pages_read_engagement
     - Klik "Generate Access Token"
     - Login dan klik "Izinkan" untuk semua permission
     - SALIN token yang muncul (ini token sementara, berlaku 1-2 jam saja)
     - Berikan token ini ke Developer segera

  5. Developer Akan Memproses Sisanya
     Developer akan menggunakan token sementara untuk mendapatkan:
     - Long-lived Token (berlaku 60 hari, untuk Instagram)
     - Page Access Token (tidak pernah expire, untuk Facebook)
     - Instagram Business Account ID
     - Facebook Page ID

DATA YANG HARUS DISERAHKAN KE DEVELOPER:
  [x] App ID
  [x] App Secret
  [x] Token sementara dari Graph API Explorer (segera, karena berlaku 1-2 jam)
  [x] Nama Facebook Page yang ingin dipantau

CATATAN:
  - Facebook Page Token tidak pernah expire (aman, tidak perlu refresh)
  - Instagram Token berlaku 60 hari, sistem akan refresh otomatis
  - Pastikan akun Instagram SUDAH terhubung ke Facebook Page sebelum mulai


================================================================================
  BAGIAN C: YouTube
================================================================================

SYARAT:
  - Punya akun Google yang memiliki YouTube Channel

APA YANG HARUS DILAKUKAN:

  1. Buat Project di Google Cloud Console
     - Buka: https://console.cloud.google.com/
     - Login dengan akun Google yang punya YouTube Channel
     - Klik dropdown project di atas > "New Project"
     - Isi nama: "UGC Portal"
     - Klik "Create"

  2. Aktifkan YouTube API
     - Di sidebar kiri, klik "APIs & Services" > "Library"
     - Cari: "YouTube Data API v3"
     - Klik "Enable"

  3. Buat Credential OAuth
     - Klik "APIs & Services" > "Credentials"
     - Klik "Create Credentials" > "OAuth client ID"
     - Jika diminta setup "OAuth consent screen":
         a. Pilih "External"
         b. Isi nama app, email support, email developer
         c. Di bagian "Scopes", tambahkan: youtube.readonly
         d. Di bagian "Test users", tambahkan email akun YouTube
         e. Simpan
     - Kembali buat OAuth client ID:
         Application type: "Web application"
         Name: "UGC Portal YouTube"
     - Klik "Create"
     - Catat:
         Client ID     = ........................ (catat di Tabel Ringkasan)
         Client Secret = ........................ (catat di Tabel Ringkasan)

  4. Hubungkan Akun YouTube
     - Minta Developer menjalankan proses otorisasi
     - Kalian akan diminta login Google dan klik "Izinkan"
     - Setelah berhasil, Developer akan mendapat:
         Access Token   = (otomatis didapat)
         Refresh Token  = (otomatis didapat)
         Channel ID     = (otomatis didapat, contoh: UCxxxxxxxxx)

DATA YANG HARUS DISERAHKAN KE DEVELOPER:
  [x] Client ID
  [x] Client Secret
  [x] Akses login ke akun Google/YouTube (untuk proses otorisasi)

CATATAN:
  - Token YouTube berlaku 1 jam, tapi sistem refresh otomatis
  - Refresh Token tidak pernah expire (kecuali kalian hapus akses manual)
  - Saat otorisasi, PENTING: klik "Izinkan" pada SEMUA permission yang diminta


================================================================================
  BAGIAN D: LinkedIn
================================================================================

SYARAT:
  - Punya akun LinkedIn dengan akses Admin ke Company Page

APA YANG HARUS DILAKUKAN:

  1. Buat Aplikasi di LinkedIn Developer
     - Buka: https://www.linkedin.com/developers/apps
     - Klik "Create App"
     - Isi:
         App name     : "UGC Portal Analytics"
         LinkedIn Page: pilih Company Page perusahaan
         App logo     : upload logo
     - Centang Legal Agreement
     - Klik "Create App"

  2. Aktifkan Fitur yang Dibutuhkan
     - Di halaman app, buka tab "Products"
     - Request akses untuk:
         "Share on LinkedIn"
         "Sign In with LinkedIn using OpenID Connect"
         "Marketing Developer Platform" (untuk statistik company page)
     - CATATAN: Marketing Developer Platform mungkin butuh waktu
       untuk di-approve oleh LinkedIn (beberapa hari)

  3. Catat Data Aplikasi
     - Buka tab "Auth"
     - Catat:
         Client ID     = ........................ (catat di Tabel Ringkasan)
         Client Secret = ........................ (catat di Tabel Ringkasan)

  4. Dapatkan Organization ID
     Cara paling mudah:
     - Buka LinkedIn Company Page di browser
     - Lihat URL di address bar, contoh:
         https://www.linkedin.com/company/12345678/
     - Angka di URL (12345678) adalah Organization ID
     - Catat:
         Organization ID = ........................ (catat di Tabel Ringkasan)

  5. Hubungkan Akun LinkedIn
     - Minta Developer menjalankan proses otorisasi
     - Kalian akan diminta login LinkedIn dan klik "Izinkan"
     - Setelah berhasil, Developer akan mendapat:
         Access Token   = (otomatis didapat)
         Refresh Token  = (otomatis didapat, jika fitur ini di-approve)

DATA YANG HARUS DISERAHKAN KE DEVELOPER:
  [x] Client ID
  [x] Client Secret
  [x] Organization ID (angka dari URL Company Page)
  [x] Akses login ke akun LinkedIn Admin (untuk proses otorisasi)

CATATAN:
  - Token LinkedIn berlaku 60 hari
  - Refresh Token berlaku 1 tahun (jika tersedia)
  - Jika Refresh Token tidak tersedia, perlu otorisasi ulang setiap 60 hari

================================================================================
  TIPS KEAMANAN
================================================================================

  - JANGAN share Client Secret, App Secret, atau Token ke orang lain
    selain Developer yang ditunjuk
  - JANGAN simpan data credential di chat group atau email tanpa enkripsi
  - Gunakan channel yang aman (misalnya: password manager, direct message
    yang terenkripsi) untuk mengirim data ke Developer
  - Jika merasa credential bocor, segera hubungi Developer untuk di-reset
