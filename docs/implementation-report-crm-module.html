01 Ringkasan Eksekutif

1.1 Overview

CRM Module pada UGC Business Command Portal adalah sistem Customer Relationship Management terintegrasi yang dirancang khusus untuk PT Utama Globalindo Cargo. Sistem ini mengelola seluruh siklus penjualan dari lead generation hingga deal closing, dengan fitur tracking aktivitas, sales planning, dan analytics dashboard.

3.3 Lead Claim Workflow

Ketika sales team claim lead dari Lead Bidding:

  1. Lead status berubah: claim_status = 'claimed'
  2. Auto-create Account dengan status calon_account
  3. Auto-create Opportunity dalam stage Prospecting
  4. Lead di-link ke Account dan Opportunity yang baru dibuat
  5. Lead muncul di "My Leads" dengan pipeline tracking

  • Priority (High/Medium/Low)

4.4 Retry/Winback Feature

Untuk account dengan status failed_account atau lost_account:

  • API endpoint: POST /api/crm/accounts/[id]/retry-prospect
  • Increment retry_count untuk tracking attempt
  • Create new opportunity dengan attempt_number baru
  • Track via Sales Plan type: winback_lost

06 Sales Plan

6.1 Tiga Jenis Sales Plan

TypeTargetObjective
maintenance_existingActive/Passive AccountsMaintain relationship, prevent churn
hunting_newNew ProspectsAcquire new customers
winback_lostLost/Failed AccountsRecover former customers

6.2 Sales Plan Workflow

1. PLANNING: Select type ‚Üí Set target company ‚Üí Choose date & method ‚Üí Add notes ‚Üì 2. EXECUTION: Record actual activity ‚Üí Capture evidence ‚Üí Log location ‚Üì 3. ASSESSMENT (Hunting only): Mark as Potential or Not Potential ‚Üì 4. CONVERSION: Create Lead ‚Üí Create Account ‚Üí Create Opportunity

6.3 Plan Fields

Planning Phase

  • plan_type (maintenance/hunting/winback)
  • company_name
  • pic_name, pic_email, pic_phone
  • planned_date
  • planned_activity_method
  • plan_notes

Execution Phase

  • realized_at (timestamp)
  • actual_activity_method
  • method_change_reason (if different)
  • realization_notes
  • evidence_url, evidence_file_name
  • location_lat, location_lng, location_address

6.4 Potential Assessment (Hunting)

StatusKondisiNext Action
pendingBelum dinilaiExecute plan & assess
potentialProspek menunjukkan interestCreate Lead/Account/Opportunity
not_potentialTidak interestedRecord reason, archive

Not Potential Reasons

  • sudah_punya_vendor - Already has vendor
  • tidak_butuh - No need currently
  • budget_tidak_ada - No budget
  • tidak_tertarik - Not interested
  • lainnya - Other (with notes)

6.5 Auto-Creation from Sales Plan

Ketika plan di-mark sebagai potential:

  1. Bisa create Lead baru (untuk tracking marketing)
  2. Bisa create Account baru (calon_account)
  3. Bisa create Opportunity dari account (Prospecting stage)
  4. Semua linked via source_account_id untuk tracking origin
